# Лабораторная работа №2

## Тема работы

Разработка GUI-приложения для учета студентов и результатов их экзаменов с возможностью поиска, удаления, постраничного просмотра и обмена данными в формате XML.

## Назначение системы

Разработанная система предназначена для хранения и обработки информации о студентах учебной группы. Для каждого студента сохраняются ФИО, номер группы и список экзаменов с оценками. Приложение позволяет добавлять новые записи, выполнять выборку по нескольким критериям, удалять подходящие записи, просматривать результаты постранично и сохранять или загружать данные из XML-файла.

## Основные функции

- добавление студента и списка его экзаменов;
- автоматический расчет среднего балла по экзаменам;
- поиск студентов по группе;
- поиск по предмету и диапазону баллов по этому предмету;
- поиск по диапазону среднего балла;
- массовое удаление записей по заданным условиям;
- постраничный просмотр списка студентов;
- сохранение данных в XML;
- загрузка данных из XML;
- проверка корректности пользовательского ввода.

## Предметная область и сущности

В системе используются две основные сущности:

- `Student` - студент, содержащий ФИО, группу и список экзаменов;
- `Exam` - экзамен, содержащий название предмета и балл.

На основе этих сущностей реализуется работа со списком студентов, вычисление среднего балла, фильтрация, удаление и сериализация данных.

## Архитектура приложения

Проект построен по слоям, близким к MVC:

- `model/entities` - доменные сущности `Student` и `Exam`;
- `model/repository` - репозиторий студентов в памяти;
- `model/services` - бизнес-логика поиска, удаления и пагинации;
- `model/io` - чтение и запись XML;
- `controller` - координация взаимодействия интерфейса и модели;
- `view` - главное окно, таблица, диалоги и элементы навигации;
- `utils` - валидация и преобразование пользовательских значений.

Такое разделение упрощает тестирование, развитие проекта и поддержку пользовательского интерфейса.

## Пользовательский интерфейс

Приложение реализовано на `PyQt5`. В главном окне отображается таблица со следующими данными:

- ФИО студента;
- группа;
- средний балл;
- список предметов и баллов.

Пользователю доступны действия через меню и панель инструментов:

- `Добавить`;
- `Поиск`;
- `Удаление`;
- `Сохранить`;
- `Загрузить`.

Для длинных списков реализована пагинация с переходом на первую, предыдущую, следующую и последнюю страницу, а также выбором размера страницы: `5`, `10` или `20` записей.

## Проверка корректности данных

В системе предусмотрена валидация вводимых данных:

- ФИО не может быть пустым;
- группа не может быть пустой;
- у студента должен быть хотя бы один экзамен;
- один и тот же предмет не должен повторяться у одного студента;
- балл должен находиться в диапазоне от `0` до `10`;
- при фильтрации по баллу обязательно должен быть указан предмет;
- нижняя граница диапазона не может быть больше верхней;
- при удалении должно быть задано хотя бы одно условие.

## Хранение данных

Сериализация выполнена в формате XML.

При сохранении формируется структура вида:

```xml
<students>
  <student>
    <name>Иванов Иван Иванович</name>
    <group>101</group>
    <exams>
      <exam>
        <subject>Математика</subject>
        <score>9</score>
      </exam>
    </exams>
  </student>
</students>
```

Для записи используется `xml.dom.minidom`, для чтения - SAX-парсер `xml.sax`.

## Тестирование

В проекте есть модульные тесты, покрывающие:

- поиск по группе;
- поиск по предмету и диапазону баллов;
- удаление записей;
- корректность XML round-trip;
- валидацию фильтров;
- работу преобразователей числовых значений.

## Запуск программы

Перед запуском требуется установленный пакет `PyQt5`.

```powershell
py -3 main.py
```

## Запуск тестов

```powershell
py -3 -m unittest discover -v
```

## Генерация демонстрационных данных

Для подготовки больших наборов данных можно сгенерировать XML-файлы:

```powershell
py -3 data\generate_demo_data.py
```

Скрипт создаёт файлы `demo_set_a.xml` и `demo_set_b.xml` в каталоге `data`. Эти файлы удобно использовать для демонстрации поиска, удаления и пагинации.

## Сценарии демонстрации системы

### 1. Добавление новой записи

1. Запустить приложение.
2. Выбрать действие `Добавить`.
3. Ввести ФИО студента и номер группы.
4. Добавить один или несколько экзаменов с баллами.
5. Сохранить запись.

Результат демонстрации:

- новая запись появляется в основной таблице;
- средний балл рассчитывается автоматически;
- данные сразу становятся доступны для поиска и удаления.

### 2. Демонстрация валидации

1. Открыть окно `Добавить`.
2. Попробовать сохранить студента с пустым ФИО или пустой группой.
3. Попробовать добавить два одинаковых предмета.
4. В окне `Поиск` указать диапазон балла без выбора предмета.
5. В окне `Удаление` нажать удаление без задания фильтров.

Результат демонстрации:

- система не выполняет некорректную операцию;
- пользователь получает понятное сообщение об ошибке.

### 3. Поиск студентов

1. Открыть окно `Поиск`.
2. Выполнить поиск по группе.
3. Выполнить поиск по предмету и диапазону баллов.
4. Выполнить поиск по диапазону среднего балла.
5. Нажать `Сбросить` и убедиться, что фильтры очищаются.

Результат демонстрации:

- отображаются только записи, удовлетворяющие заданным условиям;
- найденные записи можно пролистывать постранично.

### 4. Массовое удаление

1. Открыть окно `Удаление`.
2. Указать, например, группу или предмет и диапазон оценок.
3. Подтвердить удаление.

Результат демонстрации:

- удаляются все записи, подходящие под условие;
- система показывает количество удалённых записей;
- если совпадений нет, выводится отдельное уведомление.

### 5. Сохранение и загрузка XML

1. Добавить несколько студентов вручную.
2. Нажать `Сохранить` и выбрать XML-файл.
3. Перезапустить приложение или очистить рабочий набор, загрузив другой файл.
4. Нажать `Загрузить` и выбрать ранее сохранённый XML.

Результат демонстрации:

- данные восстанавливаются из файла без потерь;
- структура записей и оценки сохраняются корректно.

### 6. Демонстрация пагинации на большом наборе данных

1. Сгенерировать демонстрационные XML-файлы командой `py -3 data\generate_demo_data.py`.
2. Запустить программу и загрузить один из созданных XML-файлов.
3. Переключать размер страницы между `5`, `10` и `20`.
4. Использовать кнопки перехода `<<`, `<`, `>`, `>>`.

Результат демонстрации:

- система корректно разбивает записи на страницы;
- статусная строка и подпись пагинации показывают текущую страницу и общее число записей.

## Вывод

В ходе лабораторной работы разработано настольное приложение с графическим интерфейсом для учета студентов и экзаменационных результатов. Система поддерживает основные CRUD-операции над данными, поиск по нескольким критериям, пакетное удаление, пагинацию и обмен данными через XML. Архитектурное разделение на модель, контроллер и представление позволило сделать приложение понятным, расширяемым и пригодным для тестирования.
