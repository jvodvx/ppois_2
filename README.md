# Лабораторная работа №1

## Вариант 16. Модель парикмахерской

### Предметная область

Программная система моделирует работу парикмахерской, которая оказывает услуги по стрижке и уходу за волосами.

### Важные сущности варианта

- парикмахерская
- клиент
- парикмахер
- инструменты
- услуги
- зеркало
- стул

### Операции варианта

- запись клиента на услугу
- проведение стрижки
- проведение укладки
- консультация по уходу за волосами
- оплата услуги
- управление записями клиентов

## Реализованная модель

В системе используются фиксированные услуги:

- `Haircut`
- `Styling`
- `Consultation`

При запуске программы услуги автоматически создаются, если `salon_data.json` пустой.

Сущности проекта:

- `Salon` — агрегирует все данные парикмахерской
- `Client` — клиент
- `Hairdresser` — парикмахер
- `Tool` — инструмент
- `Service` — услуга
- `Mirror` — зеркало
- `Chair` — стул
- `Appointment` — запись клиента

## Поддерживаемые сценарии

Через CLI можно:

- добавить клиента
- добавить парикмахера
- просмотреть список услуг
- добавить инструмент
- добавить зеркало
- добавить стул
- создать запись на услугу
- завершить запись
- оплатить услугу
- посмотреть записи клиента
- отменить запись

### Завершение записи

В проекте используется единая операция завершения записи:

- для `Haircut` и `Styling` при завершении указываются использованные инструменты
- для `Consultation` при завершении вводятся текстовые рекомендации по уходу

После завершения запись получает статус `completed`.

## Статусы записи

Запись может находиться в одном из состояний:

- `booked`
- `completed`
- `cancelled`

## Хранение данных

Состояние программы сохраняется в файл `salon_data.json`.

В JSON сохраняются:

- клиенты
- парикмахеры
- инструменты
- услуги
- зеркала
- стулья
- записи

## Обработка ошибок

В проекте используются собственные исключения:

- `SalonError`
- `NotFoundError`
- `BookingError`
- `PaymentError`
- `ValidationError`
- `ScheduleError`
