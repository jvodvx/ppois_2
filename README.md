# Лабораторная работа №1

## Модель парикмахерской

### Описание программы

Программа представляет собой консольное приложение, моделирующее работу парикмахерской.
Система позволяет управлять клиентами, мастерами, услугами и записями на прием.

Пользователь может:

* добавлять клиентов
* добавлять мастеров
* добавлять услуги
* записывать клиентов на прием
* просматривать записи
* отменять записи

Программа реализована на языке **Python** с использованием **объектно-ориентированного программирования**.

# Архитектура приложения

Программа построена по **многослойной архитектуре**.

Архитектура включает следующие уровни:

```
Presentation — пользовательский интерфейс
Application — бизнес-логика
Domain — бизнес-сущности
Infrastructure — работа с файлами и хранение данных
```

# Подробное описание компонентов

## Presentation (уровень представления)

### SalonCLI

Класс `SalonCLI` реализует консольный интерфейс пользователя.

Функциональность:

* создание клиентов
* создание мастеров
* создание услуг
* запись клиента к мастеру
* просмотр записей
* отмена записи

Интерфейс реализован в виде **CLI-меню**.

---

## Application (уровень приложения)

### SalonManager

Класс `SalonManager` реализует основную бизнес-логику системы.

Функциональность:

* создание клиентов
* создание мастеров
* создание услуг
* запись клиента на услугу
* отмена записи
* проверка занятости мастера
* сохранение данных

Также класс отвечает за **генерацию уникальных идентификаторов**.

---

## Domain (уровень домена)

Данный уровень содержит основные сущности системы.

### Person

Абстрактный базовый класс для людей.

Наследники:

* Client
* Master

---

### Client

Класс клиента.

Атрибуты:

* id
* name
* phone

---

### Master

Класс мастера.

Атрибуты:

* id
* name
* services — список услуг
* schedule — расписание мастера

Методы:

* `add_service()` — добавление услуги
* `can_do_service()` — проверка выполнения услуги
* `is_available()` — проверка занятости
* `book_time()` — добавление времени записи

---

### Service

Класс услуги.

Атрибуты:

* id
* name
* duration
* price

---

### Appointment

Класс записи клиента.

Атрибуты:

* id
* client
* master
* service
* time
* status

Статус записи реализован через перечисление `AppointmentStatus`.

---

### Salon

Класс, агрегирующий все сущности системы.

Хранит:

* список клиентов
* список мастеров
* список услуг
* список записей

---

## Infrastructure (уровень инфраструктуры)

### SalonRepository

Класс `SalonRepository` отвечает за **сериализацию и десериализацию данных**.

Функциональность:

* сохранение данных в JSON
* загрузка данных из JSON
* восстановление связей между объектами

Файл хранения данных:

```
salon_data.json
```

---

# Обработка исключений

В системе используется собственная иерархия исключений.

```
SalonError
├── NotFoundError
├── BookingError
├── ValidationError
└── ScheduleError
```

Исключения используются для обработки:

* отсутствующих объектов
* ошибок записи
* неверных данных
* конфликтов расписания

---

# Сериализация данных

Данные системы сохраняются в файл:

```
salon_data.json
```

Сериализуются:

* клиенты
* мастера
* услуги
* записи

При запуске программы данные автоматически **восстанавливаются из файла**.

# Тестирование

Для проверки корректности работы системы реализованы **unit-тесты**.

Проверяются:

* создание клиентов
* создание услуг
* работа мастеров
* логика записи
* сериализация данных

Запуск тестов:

```
python -m unittest discover tests
```

# Запуск программы

Для запуска программы выполните:

